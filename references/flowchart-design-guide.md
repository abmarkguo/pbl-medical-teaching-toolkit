# PBL 教学流程图设计指南

## 概述

本文档提供 PBL 教学流程图的设计规范、SVG 生成模板和各类型流程图的设计要点，帮助教师制作专业、清晰的教学可视化材料。

---

## 一、流程图设计规范

### 1.1 通用规范

- **格式：** SVG（可缩放矢量图形）
- **画布宽度：** 680px（固定）
- **画布高度：** 根据内容自适应
- **viewBox：** `0 0 680 H`（H为实际高度）
- **坐标系：** 左上角为原点，X轴向右，Y轴向下

### 1.2 暗色主题配色方案

| 元素 | 颜色 | 色值 |
|------|------|------|
| 背景 | 深蓝灰 | #1a1a2e |
| 面板/卡片 | 深紫蓝 | #16213e |
| 主文字 | 浅灰白 | #e0e0e0 |
| 次要文字 | 灰色 | #a0a0b0 |
| 强调色1（蓝） | 亮蓝 | #4e9af1 |
| 强调色2（绿） | 翠绿 | #5cb85c |
| 强调色3（橙） | 暖橙 | #f0ad4e |
| 强调色4（红） | 珊瑚红 | #d9534f |
| 强调色5（紫） | 紫色 | #9b59b6 |
| 连接线 | 灰蓝 | #4a5568 |
| 箭头 | 浅灰 | #8a8a9a |

### 1.3 字体规范

- **主标题：** 18px，font-weight: 600
- **节点标题：** 14px，font-weight: 500
- **节点内容：** 12px，font-weight: 400
- **标注/注释：** 10px，font-weight: 400
- **字体族：** "Microsoft YaHei", "PingFang SC", sans-serif

### 1.4 节点规范

**矩形节点（步骤/环节）：**
- 圆角半径：8px
- 最小宽度：140px
- 最小高度：50px
- 内边距：12px

**圆形节点（起点/终点）：**
- 半径：25px
- 文字居中

**菱形节点（决策点）：**
- 宽度：120px
- 高度：60px
- 文字居中

**连接线：**
- 线宽：1.5px
- 箭头：marker-end
- 间距：节点间至少30px

## 二、流程图类型与设计要点

### 2.1 类型A：PBL模型流程图

**设计要点：**
- 展示选定模型的完整流程步骤
- 每步标注关键活动和预计时长
- 用颜色区分不同阶段（如：讨论阶段蓝色、自学阶段绿色、汇报阶段橙色）
- 含时间轴标识

**布局建议：**
- 垂直布局：从上到下展示步骤
- 水平布局：从左到右展示步骤（适合步骤较少时）
- 分组布局：将步骤按阶段分组

**示例结构（七步法）：**
```
[开始] → [Step1 澄清术语 5min] → [Step2 定义问题 5min]
    → [Step3 头脑风暴 15min] → [Step4 结构化整理 10min]
    → [Step5 学习目标 10min] → [Step6 自主学习 2-5天]
    → [Step7 汇报综合 60min] → [结束]
```

### 2.2 类型B：案例结构图

**设计要点：**
- 展示3幕式案例的信息释放结构
- 每幕标注：学习目标、知识锚点、引导问题数量
- 用连接线表示幕间的逻辑递进
- 标注信息量占比

**布局建议：**
- 三列布局：左中右分别代表三幕
- 每列内含：幕标题、临床信息、学习目标、知识锚点
- 列间用箭头连接，标注信息递进关系

### 2.3 类型C：教学环节流程图

**设计要点：**
- 展示完整的教学组织流程
- 含时间分配、角色分工、过渡节点
- 标注关键决策点（如：学生冷场时的应对方案）
- 含弹性时间标识

**布局建议：**
- 泳道图布局：按角色分泳道（导师/学生/小组）
- 时间轴标注：左侧标注时间节点
- 决策点用菱形节点

### 2.4 类型D：评估流程图

**设计要点：**
- 展示PBL评估的完整流程
- 区分形成性评估和总结性评估
- 含学生自评/互评/导师评价的整合路径
- 标注评估指标和反馈循环

**布局建议：**
- 分层布局：上层为评估类型，中层为评估维度，下层为结果整合
- 用不同颜色区分评估类型

### 2.5 类型E：基础-临床桥接图

**设计要点：**
- 展示基础医学知识到临床应用的桥接路径
- 含学科知识图谱和临床推理阶梯
- 标注关键桥接节点和知识整合区域
- 用箭头表示知识的流动方向

**布局建议：**
- 上中下三层布局：上层为基础医学，中层为桥接机制，下层为临床应用
- 左右展开为不同的桥接链

## 三、SVG 代码模板

### 3.1 基础SVG框架

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 680 H" font-family="Microsoft YaHei, PingFang SC, sans-serif">
  <!-- 背景 -->
  <rect width="680" height="H" fill="#1a1a2e"/>
  
  <!-- 标题 -->
  <text x="340" y="35" text-anchor="middle" fill="#e0e0e0" font-size="18" font-weight="600">
    流程图标题
  </text>
  
  <!-- 节点和连接线 -->
  ...
  
  <!-- 图例 -->
  ...
</svg>
```

### 3.2 矩形节点模板

```svg
<g>
  <rect x="X" y="Y" width="140" height="50" rx="8" fill="#16213e" stroke="#4e9af1" stroke-width="1.5"/>
  <text x="X+70" y="Y+25" text-anchor="middle" fill="#e0e0e0" font-size="14" font-weight="500">
    节点标题
  </text>
  <text x="X+70" y="Y+42" text-anchor="middle" fill="#a0a0b0" font-size="10">
    附加信息
  </text>
</g>
```

### 3.3 连接线和箭头模板

```svg
<defs>
  <marker id="arrow" markerWidth="10" markerHeight="10" refX="9" refY="3" orient="auto">
    <path d="M0,0 L0,6 L9,3 z" fill="#8a8a9a"/>
  </marker>
</defs>

<line x1="X1" y1="Y1" x2="X2" y2="Y2" stroke="#4a5568" stroke-width="1.5" marker-end="url(#arrow)"/>
```

### 3.4 图例模板

```svg
<g transform="translate(20, H-60)">
  <rect x="0" y="0" width="640" height="40" rx="6" fill="#16213e" opacity="0.8"/>
  <rect x="15" y="12" width="16" height="16" rx="3" fill="#4e9af1"/>
  <text x="40" y="25" fill="#a0a0b0" font-size="11">讨论阶段</text>
  <rect x="130" y="12" width="16" height="16" rx="3" fill="#5cb85c"/>
  <text x="155" y="25" fill="#a0a0b0" font-size="11">自学阶段</text>
  <rect x="245" y="12" width="16" height="16" rx="3" fill="#f0ad4e"/>
  <text x="270" y="25" fill="#a0a0b0" font-size="11">汇报阶段</text>
</g>
```

## 四、设计注意事项

### 4.1 可读性

- 节点内文字不超过2行
- 文字大小不小于10px
- 节点间距不小于30px
- 连接线不交叉（如必须交叉，用"桥跨"符号）

### 4.2 美观性

- 保持视觉对称和平衡
- 颜色使用不超过5种强调色
- 节点大小保持一致（同类型）
- 留白适当（四周留至少20px）

### 4.3 专业性

- 中英双语标注关键术语
- 含图例和缩写说明
- 标注时间节点和角色信息
- 参考文献标注（如适用）
